### 4. 📈 Plantilla de Informe del Performance del Trabajo
**Archivo**: `GPY2202_Exp2_S6_plantilla_informe_del_performance_del_trabajo.docx`

**Descripción**: Reporte de desempeño del proyecto comparando lo planificado vs. lo ejecutado.

**Contenido esperado**:
- **Resumen ejecutivo**: Estado general del proyecto (verde/amarillo/rojo)
- **Avance del cronograma**: % completado vs. planificado
- **Desempeño de costos**: Presupuesto gastado vs. planificado
- **Avance del alcance**: Entregables completados
- **Gestión de valor ganado (EVM)**:
  - PV (Planned Value): Valor planificado
  - EV (Earned Value): Valor ganado
  - AC (Actual Cost): Costo real
  - SPI (Schedule Performance Index): EV/PV
  - CPI (Cost Performance Index): EV/AC
- **Issues y riesgos principales**
- **Acciones correctivas tomadas**
- **Proyecciones para el próximo período**

**Aplicación al proyecto Intranet Konecta**:
- Ejemplo: Al cierre de Sprint 2 (Semana 10):
  - Avance cronograma: 45% completado (planificado 40%) → Adelante
  - Presupuesto: $48M gastados de $40M planificados → Sobre presupuesto
  - SPI = 1.125 (adelante en cronograma)
  - CPI = 0.83 (sobre presupuesto)

## 💡 CONSEJOS Y MEJORES PRÁCTICAS

### Para la Gestión de Riesgos:
- **Sé específico**: "Riesgo de retraso" es vago; "Retraso de 2 semanas en Sprint 2 por falta de APIs del sistema de RRHH" es específico
- **Categoriza bien**: Usa categorías consistentes (técnico, organizacional, externo, gestión)
- **Cuantifica**: Usa escalas numéricas claras (1-5 para probabilidad e impacto)
- **Asigna responsables**: Cada riesgo debe tener un Risk Owner
- **Revisa regularmente**: Los riesgos evolucionan, actualiza tu matriz semanalmente

### Para el Informe de Performance:
- **Usa métricas estándar**: SPI, CPI, EAC, ETC, VAC son estándares PMI
- **Visualiza con gráficos**: Un gráfico de EVM vale más que mil palabras
- **Sé honesto**: Si el proyecto está atrasado o sobre presupuesto, repórtalo
- **Propón acciones**: No solo reportes problemas, propón soluciones
- **Proyecta**: Usa tendencias actuales para estimar el final del proyecto

### Para las Solicitudes de Cambio:
- **Documenta todo impacto**: Alcance, cronograma, costo, calidad, riesgos
- **Evalúa alternativas**: Muestra que consideraste otras opciones
- **Cuantifica**: "Más tiempo" es vago; "2 semanas adicionales" es específico
- **Justifica**: Explica por qué el cambio es necesario o beneficioso
- **Sigue el proceso**: No implementes cambios sin aprobación formal

## 📚 RECURSOS ADICIONALES

### Conceptos Clave:
- **Riesgo**: Evento incierto que, de ocurrir, afecta los objetivos del proyecto
- **EVM (Earned Value Management)**: Técnica de medición del desempeño del proyecto
- **SPI (Schedule Performance Index)**: Índice de desempeño del cronograma (EV/PV)
- **CPI (Cost Performance Index)**: Índice de desempeño de costos (EV/AC)
- **Control de Cambios**: Proceso formal para gestionar cambios en la línea base

### Fórmulas Importantes:
```
SPI = EV / PV
  > 1.0 = Adelante en cronograma
  < 1.0 = Atrasado en cronograma

CPI = EV / AC
  > 1.0 = Bajo presupuesto (eficiente)
  < 1.0 = Sobre presupuesto (ineficiente)

EAC (Estimate at Completion) = BAC / CPI
VAC (Variance at Completion) = BAC - EAC
```
