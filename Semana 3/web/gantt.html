📊 Análisis del Cronograma

Ruta Crítica del Proyecto

La ruta crítica está compuesta por las siguientes tareas clave que determinan la duración mínima del proyecto:

  • Sprint 1 - Módulos Core (Semanas 7-8): Base del sistema, autenticación y estructura
  • Sprint 2 - RRHH y Comunicaciones (Semanas 8-10): Funcionalidades principales de autoservicio
  • Sprint 3 - Directorio y CMS (Semanas 11-12): Sistema de gestión de contenidos
  • Sprint 4 - Integraciones (Semanas 13-14): Conexión con sistemas legacy
  • Pruebas No Funcionales (Semanas 13-14): Rendimiento, seguridad y compatibilidad
  • Auditoría de Seguridad (Semana 14): Validación de seguridad externa
  • Pruebas UAT (Semanas 15-16): Aceptación de usuarios piloto
  • Capacitación (Semanas 17-18): Formación de 2,700 usuarios
  • Go-Live (Semana 19): Puesta en producción del sistema
